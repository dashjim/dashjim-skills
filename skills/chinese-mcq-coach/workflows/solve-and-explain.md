# Workflow: 解题教学（逐项比对 → 指认陷阱 → 给答案）

<required_reading>
**根据题型，先读对应的陷阱清单：**
- 现代文阅读 → `references/modern-text-traps.md`
- 古诗文（文言文理解 / 诗歌鉴赏）→ `references/classical-text-traps.md`
- 语言文字运用（成语 / 病句 / 标点 / 衔接 / 得体）→ `references/language-use-traps.md`

拿不准题型时，三份都扫一眼再判断。
</required_reading>

<process>
## Step 1: 确认信息完整
- 检查是否同时有【文段】【题干】【全部选项】。
- 若缺原文（尤其是"下列理解符合/不符合原文"这类题），停下来向用户索要原文，不要凭选项猜。
- **成功标准**：手上有做出判定所需的全部材料。

## Step 2: 读懂题干，明确在问什么
- 分清是选"正确项"还是选"不正确项"——这是最容易看反、导致全盘皆输的地方。
- 明确考点：理解、推断、概括、鉴赏、还是语用。
- **成功标准**：用一句话复述题目要求，并标注要选的是"对"还是"错"。

## Step 3: 逐项文本比对
对**每一个**选项：
1. 把选项拆成若干分句/信息点。
2. 每个信息点回原文定位对应文字（引用原文关键词）。
3. 判定该信息点是【有据且一致】/【有据但被扭曲】/【原文无据】。
4. 只要有一处扭曲或无据，该项即为干扰项。
- **成功标准**：每个选项都有明确的原文出处或"无据"结论，不留"感觉对"的空白。

## Step 4: 指认陷阱类型
对每个错误选项，对照 reference 指出它属于哪一类陷阱（如"强加因果""以偏概全""未然变已然"），并说明关键破绽词。
- **成功标准**：每个干扰项都贴上一个明确的陷阱标签，并指出触发词。

## Step 5: 给出答案与理由
- 明确给出答案选项。
- 用"正确项为何正确 + 每个干扰项踩了哪个陷阱"的结构收束。
- **成功标准**：结论唯一、理由可回溯到原文。

## Step 6: 顺带教学（轻量）
- 一句话总结这道题最值得记住的破解点（例如"看到'必然'先回原文核查程度"）。
- 不喧宾夺主，一两句即可。
- **成功标准**：学生下次遇到同类陷阱能自己识别。
</process>

<output_format>
建议按此结构输出（可依题目繁简调整）：

**题干**：选【正确 / 不正确】的一项。考点：___

**逐项分析**：
- A. 原文依据："..." → 判定 + 陷阱类型（若错）
- B. ...
- C. ...
- D. ...

**答案**：___

**一句话心法**：___
</output_format>

<anti_patterns>
避免：
- 没有原文就直接判断对错。
- 只给答案不指认干扰项类型（失去教学价值）。
- 看反题干（把选"不正确"做成选"正确"）。
- 用"语感""一般来说"代替原文比对。
- 过度推断，替原文脑补它没说的意思。
</anti_patterns>

<success_criteria>
- [ ] 信息完整性已确认（缺原文则已索要）
- [ ] 题干要求（正确/不正确）已明确
- [ ] 每个选项都做了原文比对，有出处或"无据"结论
- [ ] 每个干扰项都贴了陷阱标签并指出破绽词
- [ ] 给出唯一答案且理由可回溯原文
- [ ] 附一句可迁移的破解心法
</success_criteria>
